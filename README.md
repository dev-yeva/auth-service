## Overview

The `auth` directory is a gRPC server responsible for:
* user registration
* authentication
* JWT token issuance
* administrator privilege verification

The service follows a layered architecture (transport  →  business logic  →  storage) and stores user credentials in SQLite with salted, hashed passwords.

The `client` directory is a command-line tool for interacting with the server.

The `protos` directory contains the Protocol Buffer definitions and autogenerated gRPC client and server code.

## Prerequisites

[Task](https://taskfile.dev/docs/installation): a simple `make` alternative for workflow automation.

## Installation

```shell
git clone https://github.com/dev-yeva/auth-service
cd auth-service
```
## Usage

| Command | Description |
|---|---|
| `task download-deps` | Download dependencies |
| `task migrate-up` | Create database tables |
| `task migrate-down` | Delete database tables |
| `task run-server` | Run the server |
| `task run-client` | Run the client |
